## Architectural notes
- **Cylinder orientation:** the Three.js `CylinderGeometry` default axis is +Y. To align it from atom A → atom B, compute `dir = (B - A).normalize()` and use `mesh.quaternion.setFromUnitVectors(new Vector3(0, 1, 0), dir)`. Set position to the midpoint and scale Y to the bond length.
- **Reuse geometry per bond is fine for now.** Same instinct as A1 day 1 — premature optimization is not the goal. Mesh-per-bond rebuild every `draw()` matches the atom pattern.
- **No raycaster, no `hitBondTest` equivalent today.** `ui.js` keeps using the Canvas 2D path for picking. The Three.js path is "render-only" until A2 lands BVH.
- **Bond color:** Canvas 2D uses CSS variable `--ink` (lines 393–396 in renderer.js). Hard-code `0x666666` (light) and `0xaaaaaa` (dark) for now; theme reactivity can come later. Cu bonds get a slightly thicker radius (matches the `isCu` thickening in renderer.js:396).
- **Don't add bond labels (length in Å).** That's a Canvas 2D-only feature today and belongs in a CSS2DRenderer pass — defer to A1 day 4+.
- **Test mocking:** existing `jest.unstable_mockModule` setup in `renderer-three.test.js` already stubs `three`. Add `bonds: []` to the mock state shape if not already present, and verify `getAtom` mock returns objects with `x/y/z`.
