## Architectural notes

### Sphere-group shape (from `state.js::getCavitySpheres`)
Each cavity entry pushed into `getAllDrawGroups()` looks like:
```js
{ cx, cy, cz, r, isSphere: true, ids: [], color: PRESET_COL['cavities'], cavityId: <number> }
```
No `faces` / `edges` arrays — must be skipped by the polyhedra branch (already done at line 83).

### Rendering pattern to follow
The Canvas 2D renderer (`renderer.js` ~line 170) draws each cavity as two layered rings (front/back) with a translucent disk. **Do not port that.** Three.js gets the simpler, geometrically honest version: one `SphereGeometry(r, 24, 16)` mesh at `(cx, cy, cz)`, `MeshBasicMaterial` with `transparent:true`, `opacity:0.25`, `depthWrite:false`. The orientation rings were a 2D depth-cue hack — unnecessary in a real 3D scene.

### Where to put it
Inside the existing `for (const group of getAllDrawGroups())` loop in `renderer-three.js::draw()`. Replace the `if (group.isSphere) continue;` line with a branch:
```js
if (group.isSphere) {
    const geo = new THREE.SphereGeometry(group.r, 24, 16);
    const mat = new THREE.MeshBasicMaterial({
        color: new THREE.Color(group.color),
        transparent: true,
        opacity: 0.25,
        depthWrite: false,
    });
    const mesh = new THREE.Mesh(geo, mat);
    mesh.position.set(group.cx, group.cy, group.cz);
    polyGroup.add(mesh);
    continue;
}
```
Adding to `polyGroup` reuses the existing rotation sync at the bottom of `draw()` — no new group needed.
