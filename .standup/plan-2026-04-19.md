## Architectural notes
- **Parallel renderer pattern, not in-place rewrite.** `renderer.js` keeps working unchanged; `renderer-three.js` is a separate module behind a feature flag. This lets us migrate piece-by-piece (atoms Ôćĺ bonds Ôćĺ polyhedra Ôćĺ picking Ôćĺ fog Ôćĺ supercell) over multiple days without breaking existing tests.
- **No bundler.** Use CDN + ES module `<script type="importmap">` per [phase-2/prd.md](../Docs/phase-2/prd.md) non-goal #3.
- **Coordinate convention:** `state.atoms[].x/y/z` are Cartesian, centered at origin, max radius ÔëĄ 10. Use these directly as Three.js positions ÔÇö no transform needed.
- **Don't touch `ui.js`.** Mouse handling, picking, gizmo drag all stay on the Canvas2D path until A1 lands fully. Today is "render-only".
- **Test mocking:** Three.js needs a `WebGLRenderingContext`. If `jest.setup.js` doesn't already stub one, add a minimal `getContext('webgl')` mock returning `{}` so the module loads without crashing in jsdom. The test only verifies "module loads + `init` doesn't throw" ÔÇö not actual rendering.
